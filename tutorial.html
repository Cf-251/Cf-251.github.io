Vivid button hover effects

Looking for some bright button hover effects? Then you're on the right track. In this tutorial we'll discover how three different button hover effects work. They'll definitely make your website more stylish and fresh-looking.

We'll go through the process of creating three buttons, starting with markup and ending with their usage and customization. You can try to hover the buttons below in Codepen demo to see what we are going to talk about.

See the Pen Vivid button hover effects by Helen (@CF-251) on CodePen.

So let's start from the initial set-up:

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <title>Buttons</title>
    <link rel="stylesheet" href="button-styles.css">
</head>
<body>

<section>
    <div class="wrap">
        <a href="#" class="btn btn-md btn-hover-1">Button</a>
        <a href="#" class="btn btn-md btn-hover-2"><span>Button</span></a>
        <a href="#" class="btn btn-md btn-hover-3"><span>Button</span></a>
    </div>
</section>

</body>
</html>

You should note that text in two last buttons is placed into span element to make hover effect work properly.
Now let's apply some general styles and sizing to buttons.

// Button General Styles
.btn {
  display: inline-block;
  text-align: center;
  border-radius: 3px;
  border: 1px solid;
  cursor: pointer;
  position: relative;
  text-decoration: none;
  transition: .5s all ease;
}

// Button Layout Styles
.btn-md {
  padding: 20px 50px;
  font-size: 20px;
  line-height: 1.2;
}

  1. Button #1 hover effect

    First hover effect is based on using :before and :after pseudo-elements, which are set in bottom left and right corners and have no width and height, they are made by using border properties. The width of borders grows to 200 px while hovering and they form the triangles overlapping each other.

    Keep im mind that if your button becomes bigger (for example the button text is longer), the border-width value should also be increased. 
    // Button Layout Styles
.btn-md {
  padding: 20px 50px;
  font-size: 20px;
  line-height: 1.2;
}

// Button #1 hover effect
.btn-hover-1 {
  color: #fff;
  position: relative;
  overflow: hidden;
  transition: 0.5s;

  &:before,
  &:after {
    content: '';
    width: 0;
    height: 0;
    border: 0 solid;
    position: absolute;
    transition: 0.5s;
    z-index: -1;
  }

  &:before {
    border-color: transparent transparent transparent #ace2e0;
    bottom: 0;
    left: 0;
  }

  &:after {
    border-color: transparent transparent #ace2e0 transparent;
    right: 0;
    bottom: 0;
  }

  &:hover {
    color: #3B3C40;
    border-color: #ace2e0;

    &:before {
      border-width: 200px 0 0 200px;
    }

    &:after {
      border-width: 0 0 200px 200px;
    }
  }
}

  2. Button #2 hover effect

    In this hover effect we use pseudo-elements not only for button itself, but for the span element which is inside, too. Each of them creates a block with 1/4 width of the button and zero height, which grows to 100% on hover.

    // Button #2 hover effect
.btn-hover-2 {
  color: #fff;
  position: relative;
  overflow: hidden;
  transition: 0.3s;

  &:before, &:after,
  span:before,
  span:after {
    content: '';
    position: absolute;
    top: 0;
    width: 25%;
    height: 0%;
    background: #8c1220;
    transition: 0.3s;
    z-index: -1;
  }

  &:before {
    left: 0;
  }

  &:after {
    left: 50%;
  }

  span {
    &:before, &:after {
      top: auto;
      bottom: 0;
    }

    &:before {
      left: 25%;
    }

    &:after {
      left: 75%;
    }
  }

  &:hover {
    border-color: #8c1220;
    &:before, &:after,
    span:before, span:after {
      height: 100%;
    }
  }
}

    Pretty simple, right? Let's go further.

  3. Button #3 hover effect

    The same principle is used in the third hover effect - four pseudo-elements cover the button increasing their height on hover. But the most interesting goes next. We use 0.5 transparency value on the background to get this semi-transparent effect when first two pseudo-elements overlay the button. After the transition-delay within 0.4s another two elements (span:before and span:after) do the same thing and it looks like they completely fill the button background.

    // Button #3 hover effect
.btn-hover-3 {
  color: #fff;
  position: relative;
  overflow: hidden;
  transition: 0.5s;

  &:before,
  &:after,
  span:before,
  span:after {
    content: '';
    background: rgba(221, 213, 146, 0.4);
    position: absolute;
    top: 0;
    left: 0;
    transition: 0.4s;
    z-index: -1;
    width: 100%;
    height: 0%;
  }

  &:after,
  span:before {
    top: auto;
    bottom: 0;
  }

  span:before,
  span:after {
    transition-delay: 0.4s;
  }

  &:hover {
    color: #222;
    border-color: #ddd592;
    span:before,
    span:after,
    &:before,
    &:after {
      height: 100%;
    }
  }
}

You've probably noticed that the value of height property (in buttons #2 and #3) has the '%' symbol, which is not compulsory. It's applied to make the hover work properly in IE, so now all of these button effects have a cross-browser compatibility.

By the way the styles are quite easy to customize. You can change, for example, horizontal overlap in the last button to vertical changing the initial position of pseudo-elements and their width.
